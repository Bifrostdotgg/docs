The applications plugin lets you create forms that members fill in and submit for staff review. Use it for staff applications, event sign-ups, allowlist requests, or any situation where you need structured information from members.

## How applications work

You create a form in the dashboard with as many questions as needed. Members access the form by running `/application` and selecting the form they want to apply for. After submitting, their responses appear in the dashboard for staff to review, approve, or decline. Members are notified of the decision by DM.

## Creating a form

Navigate to **Applications** in the dashboard and click **New Form**.

| Setting | Description |
| --- | --- |
| **Name** | The form's display name, shown to members when they select it |
| **Description** | A short explanation of what the form is for |
| **Channel** | Where submission notifications are posted for staff |
| **Questions** | The fields members fill in (short text, long text, or multiple choice) |
| **Review roles** | Which roles can approve or decline submissions |
| **Acceptance message** | DM sent to the member when their application is approved |
| **Rejection message** | DM sent to the member when their application is declined |

### Question types

| Type | Description |
| --- | --- |
| **Short text** | A single-line text input, up to 256 characters |
| **Long text** | A multi-line text input, up to 4000 characters |
| **Multiple choice** | The member selects one option from a defined list |

## Submitting an application

```text
/application list
/application post <form>
```

`/application list` shows the member all available forms. `/application post` opens the selected form as an interactive modal in Discord. The member fills in each field and submits. If the form has more fields than fit in a single Discord modal, the submission is split across multiple steps.

## Reviewing submissions

Navigate to **Applications** in the dashboard. Each submission shows all question responses, the applicant's Discord username, and the submission timestamp. From the review panel:

- **Approve** — marks the submission approved and sends the acceptance DM to the applicant
- **Decline** — marks the submission declined and sends the rejection DM
- **Pending** — no action taken yet, visible to all reviewers

## Required permissions

| Action | Permission key |
| --- | --- |
| Submitting an application | `applications.open` (default allow) |
| Viewing form submissions | `applications.review` |
| Approving and declining submissions | `applications.manage` |
| Managing forms | `applications.manageForms` |
