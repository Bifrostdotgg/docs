The games plugin provides a full suite of interactive games — board games, casino games, and an economy system — all playable directly inside Discord. Each game is its own slash command. The economy layer gives members a virtual currency they earn through daily claims, game winnings, and tipping.

## Enabling games

Navigate to **Games > Configuration** in the dashboard. Games are enabled by default for all members once the plugin is active.

## Access control

You can restrict which channels and roles have access to games:

- **Channels** — Navigate to **Games > Configuration > Channels** and specify the channels where games are allowed. If no channels are configured, games work everywhere.
- **Roles** — Navigate to **Games > Configuration > Openers** to restrict game commands to specific roles.

Restricting games to a dedicated channel keeps them from cluttering other parts of your server.

## Economy configuration

Navigate to **Games > Configuration > Economy** to configure:

| Setting | Description |
|---|---|
| **Currency name** | What your server calls its currency (e.g. "coins", "gold", "credits") |
| **Currency emoji** | An optional emoji displayed alongside balances |
| **Daily claim amount** | How many coins members receive per daily claim |
| **Daily claim cooldown** | How long between daily claims (default 24 hours) |
| **Starting balance** | How many coins new members start with |

## Visibility settings

Individual games can be hidden from the command list without being disabled. Navigate to **Games > Configuration > Visibility** to toggle which games appear to members.

## Player stats

Navigate to **Games > Players** in the dashboard to view all members who have interacted with the games system. For each player you can see their current balance, total games played, win/loss record across each game type, and the last time they were active.

Staff with the `games.manage` permission can adjust a player's balance, reset their statistics, or remove them from the leaderboard from this page.

## Transactions

Navigate to **Games > Transactions** to view the full ledger of economy transactions across your server. Each entry records:

| Field | Description |
|---|---|
| **Player** | Which member the transaction belongs to |
| **Type** | The source of the transaction (daily claim, game win/loss, tip, admin adjustment, vote reward) |
| **Amount** | How many coins were added or deducted |
| **Balance after** | The player's balance after the transaction |
| **Timestamp** | When the transaction occurred |

Filter by player, transaction type, or date range. The transaction log is read-only.

## Analytics

Navigate to **Games > Analytics** for aggregate statistics across your server's games:

- Most played games by session count
- Total coins in circulation
- Daily active players over time
- Top earners and top spenders

## Trivia questions

If the trivia game is enabled, navigate to **Games > Trivia Questions** to manage the question bank. You can:

- Add custom trivia questions with multiple-choice answers
- Import a batch of questions from a CSV file
- Edit or delete existing questions
- Enable or disable individual questions without deleting them

Custom questions appear in the trivia rotation alongside any built-in questions.

## Vote rewards

Navigate to **Games > Vote Rewards** to configure economy rewards for members who vote for your server on top.gg. When a member votes, Heimdall receives a webhook notification from top.gg and automatically credits the configured coin reward to their balance.

To set up vote rewards:

1. Enter your top.gg server URL in the configuration
2. Copy the webhook URL provided and paste it into your top.gg server listing's webhook settings
3. Set the reward amount (coins per vote)
4. Optionally configure a bonus multiplier for weekend votes

## Required permissions

| Action | Permission key |
|---|---|
| Playing games | `games.use` (default allow) |
| Managing configuration | `games.manage` |
